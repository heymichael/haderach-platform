Test Lineup

Selection: haderach platform baseline

Trigger: deploy-linked + scheduled + canary

Environment: staging first, then production monitor

Platforms: firebase-hosting, docs-shell routes

Deploy Smoke (Blocking)

  • Root route reachability (`/`).
  • Docs shell route reachability (`/docs`).
  • Requirements catalog loads and project deep-link resolves.
  • Priorities and Architecture tabs render expected content.

Nightly E2E Regression (Non-Blocking)

  • Tab navigation + URL state persistence for `tab` and item params.
  • Catalog/list/detail transitions for requirements and testing tabs.
  • Fallback handling for missing/invalid catalog slugs.
  • Generated docs integrity after regeneration and sync workflows.

Prod Monitor (Non-Blocking)

  • Route health for `/`, `/docs`, and promoted app docs routes.
  • Header/indexing policy checks (`X-Robots-Tag`, noindex/noarchive).
  • Basic shell marker checks for critical docs pages.
